# Task 1: DB設計

## 概要
テーブル設計とAPIエンドポイント仕様を定義する。

---

## 作業内容

### 1. テーブル設計
以下のテーブルを設計し、カラム定義・型・制約・テーブル間のリレーションをドキュメント化する。

#### 入力

以下ファイルを読み込むこと

- docs/spec/requirements.md

#### 出力

markdownファイルにmermaid形式で出力する
DDLファイルを作成する

- docs/spec/db_structure.md
- src/main/resources/db/migration/V1__create_tables.sql


### 2. APIエンドポイント仕様定義
以下のエンドポイントについてメソッド・パス・リクエスト/レスポンスの項目を定義する。

- 著者登録
- 著者更新
- 書籍登録
- 書籍更新
- 著者に紐づく書籍一覧取得

#### 入力

以下ファイルを読み込むこと

- docs/spec/requirements.md
- docs/spec/db_structure.md（Task 1-1 の成果物）

#### 出力

各エンドポイントについて以下の項目をMarkdownで定義する

- メソッド・パス
- パスパラメータ（名前・型・説明）
- クエリパラメータ（名前・型・必須/任意・説明）
- リクエストボディ（フィールド名・型・必須/任意・バリデーション条件）
- レスポンスボディ（フィールド名・型・説明）
- エラーレスポンス（ステータスコード・発生条件）

出力先:

- docs/spec/api_spec.md
